# gitignore 模式规范

本文档描述 `.gitignore` 文件的模式匹配规则，基于 [git 官方文档](https://git-scm.com/docs/gitignore)。

## 1. 概述

`.gitignore` 文件指定 Git 应该忽略的未跟踪文件。已跟踪的文件不受影响。

### 忽略文件来源（优先级从高到低）

1. 命令行指定的模式
2. 当前目录的 `.gitignore` 文件
3. 上级目录的 `.gitignore` 文件（更近的优先）
4. `$GIT_DIR/info/exclude` 文件
5. `core.excludesFile` 配置指定的文件

同一文件内，**最后匹配的模式优先**。

## 2. 格式规则

### 空行

空行不匹配任何文件，可用作分隔符增强可读性。

### 注释

`#` 开头的行为注释。使用 `\#` 匹配字面的 `#`：

```
\#file    # 匹配名为 #file 的文件
```

### 尾部空格

行尾的空格默认被忽略。使用 `\ ` 匹配字面空格：

```
foo\      # 匹配名为 "foo " 的文件（末尾带空格）
```

### 尾部反斜杠

模式以 `\` 结尾时，该模式无效，永不匹配任何文件：

```
foo\      # 无效模式（注意：这是行尾的 \，不是转义空格）
```

## 3. 通配符

### `*` — 匹配任意字符（除 `/`）

`*` 匹配零个或多个字符，但不能跨越目录边界（不匹配 `/`）：

| 模式 | 匹配 | 不匹配 |
|------|------|--------|
| `*.o` | `foo.o` | `dir/foo.o` |
| `foo*` | `foo`, `foobar` | `foo/bar` |
| `*/*.c` | `dir/foo.c` | `dir/sub/foo.c` |

### `?` — 匹配单个字符（除 `/`）

`?` 匹配恰好一个字符，但不匹配 `/`：

| 模式 | 匹配 | 不匹配 |
|------|------|--------|
| `f?o` | `foo`, `fbo` | `f/o`, `fo` |
| `??` | `ab` | `a`, `abc` |

### `[...]` — 字符类

匹配方括号内的任一字符：

- `[abc]` — 匹配 `a`、`b` 或 `c`
- `[a-z]` — 匹配 `a` 到 `z` 范围内的字符
- `[0-9a-fA-F]` — 匹配十六进制字符
- `[!abc]` 或 `[^abc]` — 取反，匹配除 `a`、`b`、`c` 外的字符

**注意**：方括号表达式不能匹配 `/`，即使在方括号内显式包含 `/`。

### `**` — 双星号

双星号仅在以下位置有特殊含义：

#### `**/foo` — 匹配任意深度的 `foo`

```
**/foo       # 匹配 foo, a/foo, a/b/foo
```

#### `abc/**` — 匹配 `abc/` 下所有内容

```
abc/**       # 匹配 abc/x, abc/x/y/z
```

#### `a/**/b` — 匹配零或多个中间目录

```
a/**/b       # 匹配 a/b, a/x/b, a/x/y/b
```

#### 非正确分隔

当 `**` 未被 `/` 或模式边界正确分隔时，视为两个独立的 `*`：

```
a**b         # 等价于 a**b，每个 * 匹配除 / 外的任意字符
```

## 4. 取反 `!`

`!` 前缀重新包含已排除的文件：

```
*.o          # 忽略所有 .o 文件
!foo.o       # 但保留 foo.o
```

使用 `\!` 匹配字面的 `!`：

```
\!file       # 匹配名为 !file 的文件
```

### 关键限制

**如果父目录被排除，则无法重新包含其子文件。** Git 不会扫描被排除目录的内容，因此其中的取反模式不会被评估：

```
dir/         # 忽略 dir 目录
!dir/file    # 无效！无法重新包含 dir/file
```

这是因为 Git 遇到 `dir/` 时跳过整个目录，不会再检查 `dir/file` 是否匹配取反模式。

## 5. 目录模式

### `foo/` — 仅匹配目录

尾部 `/` 表示该模式仅匹配目录，不匹配同名文件：

| 模式 | 匹配文件 `foo` | 匹配目录 `foo` |
|------|---------------|---------------|
| `foo` | 是 | 是 |
| `foo/` | 否 | 是 |

两种形式匹配到目录时，都会排除目录的全部内容。

## 6. 锚定模式

### 含 `/` 的模式（锚定）

如果模式包含 `/`（前导或中间），该模式锚定到 `.gitignore` 文件所在的目录层级：

```
/foo         # 仅匹配根目录下的 foo
foo/bar      # 仅匹配根目录下的 foo/bar
/doc/frotz   # 仅匹配根目录下的 doc/frotz
```

注意：`/doc/frotz` 和 `doc/frotz` 是等价的。前导 `/` 在已有中间 `/` 时是冗余的。

### 不含 `/` 的模式（非锚定）

如果模式不包含 `/`，则可在任意深度匹配（对 basename 进行匹配）：

```
foo          # 匹配任意深度的 foo: foo, a/foo, a/b/foo
```

### `**` 与锚定的交互

`**/foo` 含有 `/`，因此是锚定模式。但由于 `**` 匹配任意深度的前缀，效果等同于非锚定的 `foo`：

```
**/foo       # 匹配任意深度的 foo（与 foo 等价）
**/foo/bar   # 匹配任意深度下的 foo/bar
```

## 7. 反斜杠转义

`\` 用于转义特殊字符，使其变为字面量：

| 模式 | 含义 |
|------|------|
| `\*` | 字面的 `*`（不是通配符） |
| `\?` | 字面的 `?`（不是通配符） |
| `\#` | 字面的 `#`（不是注释） |
| `\!` | 字面的 `!`（不是取反） |
| `\[` | 字面的 `[`（不是字符类） |
| `\\` | 字面的 `\` |

## 8. 大小写

匹配是否区分大小写由 `core.ignoreCase` 配置决定：

- Linux（ext4 等）：默认区分大小写
- macOS（APFS）和 Windows（NTFS）：默认不区分大小写

## 9. 优先级规则

### 多文件优先级

当多个 `.gitignore` 文件匹配同一文件时：

- **更近的**（更深层目录中的）`.gitignore` 优先
- 每个文件的匹配都是相对于该文件所在目录的

### 单文件内优先级

同一 `.gitignore` 文件内：

- **最后匹配的模式生效**
- 模式按顺序从上到下评估
- 取反模式可以覆盖之前的排除

示例：

```
*.o          # 排除所有 .o 文件
!foo.o       # 保留 foo.o
foo.o        # 再次排除 foo.o（最后匹配，foo.o 被忽略）
```

## 10. 常见示例

```
# 忽略所有 .o 和 .a 文件
*.[oa]

# 忽略所有 .c 文件，但保留 foo.c
*.c
!foo.c

# 忽略 build 目录
build/

# 忽略根目录下的 TODO，但不忽略子目录中的
/TODO

# 忽略 doc 目录下的所有 .pdf 文件
doc/**/*.pdf

# 忽略所有 .log 文件（任意深度）
*.log

# 保留 important.log
!important.log
```
